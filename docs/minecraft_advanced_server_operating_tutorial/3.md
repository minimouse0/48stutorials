## GitHub
什么是开源？当开发者开发好软件时，可以将源代码也同时公开出去，这个行为就是开源。与之相反地把源代码保密，只公布软件就是闭源。平时我们日常使用的各种软件，像windows，QQ微信支付宝这些都属于闭源软件，因为它们的开发者没有公开源代码。而我们开服使用的软件大多数都是开源代码。开源和闭源各有利弊：闭源软件的使用者无法获得源代码，他们就无法自行修改软件，只能从开发者处获取，这样，开发者就可以在软件中加入各种验证机制，要求用户按照他指定的方式使用软件，例如付费等。而且由于用户接触不到源代码，他们就无法得知软件是怎么样开发出来的，这样开发者就可以在软件中放心地加入一些自己想保密的开发技术。如果是开源软件，虽然不利于付费使用和采用独家开发技术，但是开发者可以邀请志愿者一同开发，而且他的源代码也可以给社区学习，大家共同钻研同一种技术，效率更高，更利于技术的发展。  
GitHub的主要目的就是给开源软件开发者提供一个平台，让他们能够公布自己的代码并且和志愿者们合作开发。GitHub是全球最著名的开源平台和社区，绝大多数个人或小型团队的开源软件的开发者们都会将自己的开源软件发布到这个平台。  
由于GitHub是开放的外国网站，网站上时不时会有一些不能在此讨论的内容，所以运营商会刻意地屏蔽这个网站，在很多时候都不能正常访问。具体的表现如下：只要Github域名指向任意IP，该IP的443端口就会超时3分钟（https://blog.csdn.net/weixin_43659597/article/details/118882176 ）。但是我们有很多资源还必须在这个网站上获取，你就需要一些工具来辅助。目前有很多工具可以使用，如果你有那种工具，那么直接使用那种工具即可访问。还有其他工具可以使用，圈内常用的有：watt toolkit（原名steam++）、kgithub等。下面简要介绍这些工具的使用方法。   
watt toolkit：前往steampp.net下载安装软件，随后在左侧切换到网络加速，点击平台加速（免费），往下翻勾上GitHub，然后点击一键加速，随后就可以正常访问GitHub了。  
kgithub：简单从互联网访问github使用的域名，将GitHub链接中的github替换为kgithub即可。  
当你能够正常访问GitHub后，就来学习一下如何使用这个网站吧。通常情况下开服只需要用到这两种界面：仓库首页和发行版。  
首先介绍仓库页面。举例，像https://github.com/fatedier/frp 这个网页就是一个仓库页面。我们只需要关注左下角的部分，上半部分是这个软件的源代码或相关资源文件，下半部分就是这个软件的介绍，通常以左上角显示“README”开始。这里一般包括了软件的安装和使用方法，或者给出了教程的获取方式。有些软件不重视这个介绍，只是几个字草草带过。但是有的软件写得非常详细，这个介绍中包含了大量的信息，如果要使用这个软件那这些介绍就是必读的。  
然后是软件的发行版页面。发行版页面有两种进入方式，一种是点击仓库右侧的Releases（不要点击下面显示的那个版本号，那个只能专门进入查看某一个版本），一般在About下面，也有可能被隐藏。另一种方式是在仓库地址后面加上/releases。比如https://github.com/fatedier/frp 这个仓库，要进入它的发行版页面，可以直接访问https://github.com/fatedier/frp/releases。  
发行版页面中，通常每个版本下面都有其对应的更新内容和下载链接。有些时候开发者没有填写更新内容，就不会显示。下载链接在每个版本的最下方，如果没有显示，点击Assets将其展开即可。  
很多时候一个版本会提供很多个安装包，这些安装包分别对应不同的功能或不同的平台。不同功能的安装包要选择哪个，在软件的教程中一般会写明。至于确认自己需要选择哪个安装包才适合自己的服务器，根据文件名即可确认。至于这些表示平台的代码都意味着什么，下文会详细讲解。  
## CPU架构：x86和ARM
虽然CPU极其复杂和智能，但是充其量它就是一个执行指令的机器。CPU根据软件的代码执行命令，程序就运行起来了。就像是招募不同国家的人干活的人一样，命令他们需要用不同的语言，而CPU也像来自“不同的国家”，拥有不同的“语言”，CPU的“国籍”就是CPU的架构，“语言”就叫指令集。作为我的世界服务器服主，我们只需要了解“x86”和“ARM”两种架构就行了。CPU的架构和指令集分类是非常复杂的，我们只需要知道“x86”和“ARM”都是一类架构的统称，也是一类指令集的统称。支持x86的软件和x86的CPU之间，通常情况下支持x86的软件就可以运行在x86的CPU上，arm的软件就可以运行在arm的CPU上，而不同架构的软件是不能运行的。  
为了让大家更快地理解，给大家举几个例子吧。  
关注数码圈的人都知道，20年苹果整了个大活，把电脑的英特尔处理器改成了自己设计的M系列处理器，之后还说什么开发者需要适配，然后自己给提供兼容层什么的。苹果之前使用的英特尔处理器，以及AMD品牌的处理器，基本上都是x86架构。x86是台式机和笔记本上最常见的架构，通常运行Windows系统的电脑的CPU就是这个架构。而后面苹果换成的那个M系列处理器就是ARM架构。这是手机和平板电脑上最常见的CPU架构，因为这种CPU主打低功耗。我们平时使用的苹果和安卓的手机和平板都是这种架构。  
x86架构是由英特尔开发的架构，最开始用于英特尔处理器。后来的AMD拿到了英特尔的授权，于是AMD的处理器也用上了x86架构，可以运行x86架构的软件了。对了，国产的兆芯因为也有x86指令集授权，所以也能运行x86架构的软件。这种架构广泛使用在电脑上，我们平时常见的台式机和大型服务器都是这个架构的CPU，像英特尔的酷睿i几，至强（比如洋垃圾），奔腾，还有AMD的锐龙这些，都是这个架构。这也是上述提到的这些处理器都能运行Windows的x86版本的原因。你用于开服的服务器应该就是这个架构，除非……
ARM架构是由ARM公司开发的架构。没错，ARM的架构名就是那个公司的名字。这个公司把自己的ARM架构授权给各芯片厂商，于是就有了ARM百花齐放的生态。ARM广泛用于手机和平板上，像苹果的A系列，高通的骁龙系列，联发科的天玑系列，海思的麒麟系列，紫光展锐的虎贲系列都是ARM架构的处理器。所以骁龙、天玑、麒麟、虎贲都能运行安卓系统。苹果的A系列是因为苹果对处理器做了特殊的加密和版权限制，才只能运行苹果的iOS系统。除了手机，还有一些电脑在采用ARM架构的处理器，像苹果从2020年年底开始推出的所有电脑，还有搭载了骁龙X系列的电脑，都是ARM架构的处理器。  
## 32位和64位
先前大家应该也听说过计算机用0和1来表示数字的事。逢二进一，这样的数字就叫二进制，作为对比，我们平时使用的数字，也就是一二三四五六七八九十那种，就是十进制，因为它逢十进一。在计算机中，二进制数字的一位数就被称之为1比特（bit）。比特是一种数据容量单位，你可以想象这样一个场景：一张纸条上只能写下2个数字，你一共能用这张纸条传达多少种不能的数字？如果是4个数字呢？允许写下的位数越多，能够记录的数字的可能性就越多。大家应该都听说过8比特吧，8比特就是8个比特，表示一共能存放八位二进制数。它可以存下00000010，00101101，11110111，但是不能存下100001101，因为它有九位数，装不下。32位的意思就是能够表示32位数。也就是说，电脑在处理数字的过程中，是把每32个比特组成一个32位数，然后去处理一个一个32位数的。64位也是同理。
你是否听说过2147483647？这是“32位有符号整数”的最大值。也就是说，如果每32位作为一个数字处理，那么它的最大就是32位。可是有一个问题，比如说4位二进制数最大是1111，换成十进制就是15，2的4次方减一，8位数换成十进制最大是255，2的8次方减一，那32位二进制数最大应该是2的32次方减一，4294967295啊，2147483647不是2的31次方减一吗？注意“32位有符号整数”这个称呼中的“有符号”，它指的是这个数字既能表示大小，又能表示正负。为了在32个0和1中塞下一个正负这个信息，就从里面划出来一个数字用来专门表示符号，0是正，1是负，这个数就是符号位，它一般位于最前端，比如正数就是000...01011，负数就是100...01011。你应该也听说过-2147483648。假如我们对2147483647的二进制数加一，也就是111...111+1，最后会得到1000...000，也就是2147483648。但是由于第一位是表示符号的，剩下的部分才表示数字，那么在这个数字中，它的符号为负，数值为0。显然没有“负零”这种说法，于是计算机中将“负零”规定为比负的2的31次方减一要小一的数，也就是负的2的31次方，2147483648。  
32位能表示的数字虽然很多，但是远远不够计算机用的，最典型的就是内存地址。电脑用32位数工作的时候，只通过一个数字表示内存地址，那么内存中就一共只能有4294967296个地址，也就是4GB。于是这就对电脑产生了一个致命的限制：32位的电脑只能安装最多4GB内存。而有了用64位数工作的电脑，内存就可以大得多。64位的最大数字要大得多，内存一共可以达到16EB。现在还没有电脑能达到这么大内存，所以目前完全不用担心内存的限制了。  
虽然有了64位的这一机制，但是电脑的软件和硬件都要支持这个机制才行。64位的CPU在很早就出现了，但是电脑的操作系统不是。Windows系统的32位就是一个例子，32位的windows系统既可以运行在32位的硬件上也可以运行在64位的硬件上，这是因为很多64位的硬件都可以支持32位的软件运行。但是虽然64位的硬件跑起来32位的系统了，它却不能运行64位的软件，因为系统是32位的，这一点成了瓶颈。因此我们可以见到很多软件都提供了32位的版本，因为这些软件需要保证自己能够在32位的电脑上运行。好在主流64位系统也支持运行32位的软件，所以如果用了64位的操作系统，就不用那么担心软件在32位和64位上的兼容性了。  
要检查自己服务器是32位还是64位，Windows系统可以进入此电脑（win8叫这台电脑，win7和xp叫我的电脑），右键空白处点击属性，在系统的系统类型下会有一个“xx位操作系统，基于xxx的处理器”。Linux系统可以执行命令getconf LONG_BIT。
